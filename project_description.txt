Description
Modern hardware verification relies heavily on rigorous testing, a process that can be significantly accelerated using Large Language Models (LLMs). This project focuses on moving beyond basic, unpredictable AI prompting by engineering a highly structured, graph-based LLM workflow. Students will design and implement a Python-based pipeline where complex hardware testing Standard Operating Procedures (SOPs) are broken down into distinct, interconnected execution nodes. By routing the LLM's reasoning and code generation through controlled paths, conditional logic, and feedback loops, the project aims to systematically generate, refine, and validate HDL testbenches with high reliability.

Proficiency
A solid foundation in Python is essential, as the core tasks involve programming data pipelines, state management, and API integrations. Familiarity with generative AI concepts, specifically how LLMs process context and follow instructions, is required to design effective workflow nodes. While prior experience with hardware design (Verilog/VHDL) or verification methodologies is highly beneficial for evaluating the generated testbenches, enthusiastic students with strong software engineering skills can acquire the necessary domain knowledge throughout the project.

Outcome
Upon completing this project, students will possess hands-on expertise in engineering deterministic and reliable AI applications. They will master the design of graph-based LLM architectures, learning how to manage execution states, define node constraints, and integrate external tools into a cohesive pipeline. Beyond software skills, participants will learn how to translate strict engineering protocols into automated, AI-driven processes, ultimately gaining valuable insight into the future of computer-aided hardware verification.
