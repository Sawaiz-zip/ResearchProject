## Project Description

As part of this topic, students will first engage with current and relevant research on the automatic generation of Verilog code using Large Language Models (LLMs). This includes analyzing relevant scientific literature in order to understand the current state of research, existing methods, and the typical challenges and limitations of such approaches.

Another focus is on understanding how Pyverilog and Icarus Verilog work. Students will study the fundamental principles, functions, and potential applications of these tools, particularly in the analysis, processing, and examination of Verilog code.

In addition, current research dealing with the use of Pyverilog for the detection of functional errors in Verilog code will be examined. The goal is to gain an overview of which approaches already exist, what types of errors can be identified, and what limitations these methods have.

Furthermore, students will learn how Verilog code can be generated from natural-language circuit descriptions with the help of LLM APIs. The emphasis here is both on the practical use of such models and on evaluating the quality and reliability of the generated results.

Finally, the focus will be on investigating how Pyverilog and/or LLM-based methods can be used for early error localization in LLM-generated Verilog. The aim is to develop or analyze approaches that make it possible to detect, narrow down, and localize errors in automatically generated Verilog as early as possible.

## Learning Outcomes / Competencies

After completing this topic, students will be able to use Large Language Models (LLMs) in a targeted way for the generation of Verilog code and to critically evaluate their potential as well as their limitations. They will understand the fundamental challenges involved in the automatic generation of hardware description code and be able to explain why generating Verilog presents particular difficulties compared with traditional software programming languages.

Students will acquire knowledge of how LLM-generated Verilog can be systematically analyzed and evaluated. They will learn to use Pyverilog as a tool for the structural and functional examination of Verilog code in order to identify potential sources of error at an early stage, analyze code components, and better assess the quality of the generated hardware description.
